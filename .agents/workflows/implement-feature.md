# Workflow: Implement Feature

1. Read the relevant ADRs and module README.
2. Identify the owning module.
3. State assumptions, ambiguities, tradeoffs, and verification plan.
4. Add or update tests in the owning module.
5. Implement the smallest change that satisfies the request.
6. Update Google-style docstrings and module README if behavior changes.
7. Update docs pages if the public contract changes.
8. Run relevant checks.
