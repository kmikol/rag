# Workflow: Add ADR

Use this when a new architecture decision is required.

1. Create `docs/adr/00N-short-title.md`.
2. Include context, decision, rationale, implications, alternatives, and review triggers.
3. If superseding an ADR, state which ADR is superseded and why.
4. Update `docs/adr/index.md`.
5. Update `docs/mkdocs.yml`.
6. Link from related architecture or module docs.

Do not modify accepted ADR decisions in place.
