# Agent Instructions

Read these files before making changes:

1. `project.md` — project purpose and non-negotiable product facts.
2. `behavior.md` — behavioral rules for assumptions, simplicity, surgical changes, and verification.
3. `architecture.md` — service boundaries and implementation contracts.
4. `coding.md` — coding, configuration, docstring, and migration rules.
5. `testing.md` — test ownership and verification expectations.
6. `documentation.md` — module documentation, MkDocs, and ADR rules.

Then choose the relevant workflow:

- `workflows/implement-feature.md`
- `workflows/add-service.md`
- `workflows/add-adr.md`
- `workflows/update-docs.md`

Before coding, use `checklists/before-coding.md`.
Before finishing, use `checklists/before-commit.md`.
